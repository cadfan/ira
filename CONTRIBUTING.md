# Contributing to ira

Thanks for your interest in contributing to ira. This is a personal project, but quality contributions are welcome.

## Before Contributing

### Contributor License Agreement

All contributions require a signed CLA. By contributing, you assign copyright of your contribution to the project owner, allowing the project to be commercially licensed. See [CLA.md](CLA.md) for the full agreement.

To sign:
1. Read [CLA.md](CLA.md)
2. Add your name, GitHub username, and date to [CONTRIBUTORS](CONTRIBUTORS)
3. Include this addition in your first pull request

### Check Existing Issues

Before starting work, check if there's an existing issue or discussion. For significant changes, open an issue first to discuss the approach.

## Making Changes

1. Fork the repository
2. Create a branch for your changes
3. Make your changes following the project style:
   - C11 standard
   - snake_case for functions and variables
   - Document public APIs in headers
4. Test your changes with iRacing
5. Commit with clear, descriptive messages

## Pull Requests

- Keep PRs focused on a single change
- Describe what the change does and why
- Reference any related issues
- Ensure the project builds without warnings

## Recognition

Contributors who have signed the CLA and had contributions merged may be listed publicly in the [CONTRIBUTORS](CONTRIBUTORS) file and acknowledged within the application. If you prefer not to be credited, let us know in your pull request.

## Questions

Open an issue for questions about contributing.
